# 建築物耐風設計規範 第一章 總則 分析報告

## 1. 結構力學核心邏輯

本章為全篇規範的基礎，不涉及具體的結構力學計算，而是建立了整個設計流程的**分類框架**和**術語體系**。其核心邏輯在於：

1.  **定義輸入參數：** 明確了所有後續計算所需的物理量（符號）及其單位。
2.  **建立分類標準：**
    *   **建築物類型分類 (Clause 1.3):** 根據開口率，將建築物分為「開放式」、「部分封閉式」、「封閉式」。此分類直接決定了第二章中採用「風壓 (p)」還是「風力 (F)」進行設計，以及內風壓係數的選取。
    *   **結構剛性分類 (Clause 1.3):** 根據基本頻率 (`fn`)，將建築物分為「普通建築物」和「柔性建築物」。此分類決定了陣風反應因子 (`G` 或 `Gf`) 的計算方法。
3.  **定義計算對象：** 明確了「主要風力抵抗系統 (MWFRS)」和「局部構件及外部被覆物 (C&C)」的區別，為後續章節的設計路徑劃分奠定了基礎。

## 2. 主要公式與標準

本章無具體計算公式，但定義了關鍵的**判斷標準**和**邊界條件**。

*   **開放式建築物定義 (Clause 1.3):** 至少兩個牆面各有 80% 以上面積為開口。
*   **部分封閉式建築物定義 (Clause 1.3):** 同時滿足以下三個條件：
    1.  `A0 > 1.10 * A0i`
    2.  `A0 > min(0.37 m², 0.01 * Ag)`
    3.  `A0i / Agi <= 0.20`
*   **封閉式建築物定義 (Clause 1.3):** 不符合上述兩者定義者。
*   **柔性建築物定義 (Clause 1.3):** `fn < 1 Hz`
*   **有效受風面積定義 (Clause 1.3):** `A = Span * max(Effective Width, Span / 3)`

## 4. 批判性審查與疑慮

1.  **部分封閉式條件的優先級：** 規範定義了「開放式」和「部分封閉式」的條件，但未明確說明當一個建築物同時滿足「開放式」和「部分封閉式」條件時應如何處理。例如，一個建築物有兩個牆面開口率為 85%，同時也滿足部分封閉式的三個條件。根據邏輯，應優先判定為「開放式」，因為其定義更為特殊。**軟體實現時，必須將「開放式」的判斷置於「部分封閉式」之前，形成 if-else if 鏈。**
2.  **「開口」的定義模糊性：** 規範定義開口為「在設計風速下，建築物表面會造成內外空氣流通之開孔（包括可能破損之外部被覆物）」。這引入了不確定性：在設計階段，如何預知哪些外部被覆物會「可能破損」？這需要工程師根據經驗判斷，軟體無法自動化處理，應作為一個**用戶輸入參數**或**可選的破壞場景**來實現。
3.  **有效受風面積的應用場景：** 該定義明確用於決定 `(GCp)` 值（局部構件及外部被覆物）。但在計算主要風力抵抗系統 (MWFRS) 時，有效受風面積的定義是否相同？規範未在此處說明，需參考後續章節。**軟體實現時，應將此計算邏輯封裝在 C&C 設計模塊內，而非全局通用。**

```python
"""
建築物耐風設計規範 第一章 總則
建築物類型及結構剛性分類器

根據 Clause 1.3 的定義，對建築物進行分類。
"""

from dataclasses import dataclass
from enum import Enum
from typing import Optional

class BuildingType(Enum):
    """建築物類型 (Clause 1.3)"""
    OPEN = "OPEN"                           # 開放式建築物
    PARTIALLY_ENCLOSED = "PARTIALLY_ENCLOSED" # 部分封閉式建築物
    ENCLOSED = "ENCLOSED"                   # 封閉式建築物

class StructuralType(Enum):
    """結構剛性類型 (Clause 1.3)"""
    RIGID = "RIGID"         # 普通建築物 (fn >= 1 Hz)
    FLEXIBLE = "FLEXIBLE"   # 柔性建築物 (fn < 1 Hz)

@dataclass
class BuildingGeometry:
    """建築物幾何與開口數據 (Clause 1.2, 1.3)"""
    # 迎風向外牆面之總面積 (m2)
    Ag: float
    # 迎風向外牆面之總開口面積 (m2)
    A0: float
    # 非迎風向之各牆面（含屋頂）總面積 (m2)
    Agi: float
    # 非迎風向之各牆面（含屋頂）總開口面積 (m2)
    A0i: float
    # 建築物基本自然頻率 (Hz)
    fundamental_frequency_n: float

    def __post_init__(self):
        """基本數據驗證"""
        if self.Ag < 0 or self.A0 < 0 or self.Agi < 0 or self.A0i < 0:
            raise ValueError("面積和開口面積不能為負數。")
        if self.fundamental_frequency_n <= 0:
            raise ValueError("基本自然頻率必須為正數。")

def classify_building_type(geo: BuildingGeometry) -> BuildingType:
    """
    根據 Clause 1.3 的定義，判斷建築物類型。

    判斷邏輯優先級: 開放式 -> 部分封閉式 -> 封閉式

    Args:
        geo: 建築物幾何與開口數據。

    Returns:
        建築物類型枚舉值。
    """
    # 1. 判斷是否為開放式建築物
    # 條件: 至少兩個牆面各有 80% 以上面積為開口。
    # 注意: 此處簡化為檢查迎風面和非迎風面，實際應用中需檢查所有牆面。
    #       假設迎風面為一個牆面，非迎風面為其餘牆面總和。
    #       更嚴謹的實現需要遍歷所有牆面。
    #       此處簡化為：如果迎風面開口率 >= 80% 且非迎風面開口率 >= 80%，則視為開放式。
    #       這是一個常見的工程簡化，但嚴格來說應檢查任意兩個牆面。
    opening_ratio_windward = geo.A0 / geo.Ag if geo.Ag > 0 else 0
    opening_ratio_leeward = geo.A0i / geo.Agi if geo.Agi > 0 else 0

    # 開放式建築物定義 (Clause 1.3): 至少兩個牆面各有 80% 以上面積為開口
    # 此處簡化為檢查迎風面和一個代表「非迎風面」的總和。
    # 在實際應用中，需要對每個牆面單獨計算開口率。
    if opening_ratio_windward >= 0.8 and opening_ratio_leeward >= 0.8:
        return BuildingType.OPEN

    # 2. 判斷是否為部分封閉式建築物
    # 條件 (Clause 1.3): 同時滿足以下三個條件
    # (1) A0 > 1.10 * A0i
    condition_1 = geo.A0 > 1.10 * geo.A0i

    # (2) A0 > 0.37 m2 或 A0 > 0.01 * Ag (取小值)
    threshold_2 = min(0.37, 0.01 * geo.Ag)
    condition_2 = geo.A0 > threshold_2

    # (3) A0i / Agi <= 0.20
    condition_3 = (geo.A0i / geo.Agi) <= 0.20 if geo.Agi > 0 else False

    if condition_1 and condition_2 and condition_3:
        return BuildingType.PARTIALLY_ENCLOSED

    # 3. 其餘情況為封閉式建築物
    return BuildingType.ENCLOSED

def classify_structural_type(geo: BuildingGeometry) -> StructuralType:
    """
    根據 Clause 1.3 的定義，判斷結構剛性類型。

    Args:
        geo: 建築物幾何與開口數據 (主要使用 fundamental_frequency_n)。

    Returns:
        結構剛性類型枚舉值。
    """
    # 柔性建築物定義 (Clause 1.3): 細長建築物之基本自然頻率 fn < 1 Hz
    if geo.fundamental_frequency_n < 1.0:
        return StructuralType.FLEXIBLE
    else:
        return StructuralType.RIGID

def calculate_effective_area(span: float, effective_width: float) -> float:
    """
    計算有效受風面積 A (Clause 1.3)。

    有效受風面積為跨距長度與有效寬度之乘積。
    有效寬度不必小於其跨距長度的 1/3。

    Args:
        span: 構件跨距長度 (m)。
        effective_width: 構件有效寬度 (m)。

    Returns:
        有效受風面積 (m2)。
    """
    # Clause 1.3: 有效寬度不必小於其跨距長度的 1/3
    adjusted_width = max(effective_width, span / 3.0)
    return span * adjusted_width

# --- 使用範例 ---
if __name__ == "__main__":
    # 範例 1: 一個典型的封閉式建築物
    geo1 = BuildingGeometry(
        Ag=100.0, A0=5.0,
        Agi=300.0, A0i=10.0,
        fundamental_frequency_n=2.0
    )
    print(f"範例 1: 建築物類型 = {classify_building_type(geo1).value}, 結構類型 = {classify_structural_type(geo1).value}")
    # 預期輸出: 範例 1: 建築物類型 = ENCLOSED, 結構類型 = RIGID

    # 範例 2: 一個部分封閉式建築物 (滿足所有條件)
    geo2 = BuildingGeometry(
        Ag=100.0, A0=20.0,  # A0 > 1.1 * A0i (20 > 11), A0 > min(0.37, 1.0) (20 > 0.37)
        Agi=300.0, A0i=10.0, # A0i/Agi = 0.033 <= 0.20
        fundamental_frequency_n=0.5
    )
    print(f"範例 2: 建築物類型 = {classify_building_type(geo2).value}, 結構類型 = {classify_structural_type(geo2).value}")
    # 預期輸出: 範例 2: 建築物類型 = PARTIALLY_ENCLOSED, 結構類型 = FLEXIBLE

    # 範例 3: 一個開放式建築物 (兩個牆面開口率 > 80%)
    geo3 = BuildingGeometry(
        Ag=100.0, A0=85.0,  # 開口率 85%
        Agi=200.0, A0i=170.0, # 開口率 85%
        fundamental_frequency_n=1.5
    )
    print(f"範例 3: 建築物類型 = {classify_building_type(geo3).value}, 結構類型 = {classify_structural_type(geo3).value}")
    # 預期輸出: 範例 3: 建築物類型 = OPEN, 結構類型 = RIGID

    # 範例 4: 有效受風面積計算
    span = 5.0
    width = 1.0
    area = calculate_effective_area(span, width)
    print(f"範例 4: 跨距={span}m, 有效寬度={width}m, 有效受風面積={area:.2f} m2")
    # 預期輸出: 範例 4: 跨距=5.0m, 有效寬度=1.0m, 有效受風面積=8.33 m2 (因為 max(1.0, 5/3) = 1.667)
```
